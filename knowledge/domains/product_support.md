# Product Support Guidelines

## Scope
This domain covers user requests related to product understanding, recommendations, compatibility, setup, and general product inquiries.

## Objectives
- Help users make informed decisions
- Provide accurate product information
- Guide users through setup and usage
- Recommend suitable products based on needs

## Common Subdomains
- product_inquiry
- product_recommendation
- product_compatibility
- peripheral_compatibility
- product_setup

## Key Principles

### 1. User-Centric Guidance
Understand:
- user needs
- use case
- constraints (budget, compatibility, etc.)

Then adapt the response accordingly.

### 2. Accuracy
Provide:
- correct specifications
- real compatibility info
- realistic expectations

Avoid assumptions if unsure.

### 3. Simplicity
Explain technical details in a simple way.
Avoid unnecessary jargon.

### 4. Actionability
Whenever possible:
- give concrete steps
- suggest clear options
- provide comparisons if relevant

## Response Style
- Friendly and helpful
- Educational but concise
- Structured (bullets, steps, comparisons)

## Recommendation Guidelines
When recommending products:
- justify the recommendation
- mention trade-offs
- avoid bias unless explicitly required

## Setup Guidance
When helping with setup:
- provide step-by-step instructions
- highlight common mistakes
- mention prerequisites

## Escalation Rules
Escalate when:
- compatibility is unclear or undocumented
- product is defective (move to technical_support)
- request requires internal data not available

## Do Not
- Recommend incompatible products
- Give outdated or unverified information
- Overwhelm the user with unnecessary detail

## Output Expectations
Responses should:
- directly answer the question
- provide useful next steps
- adapt to user intent (exploration vs decision)
