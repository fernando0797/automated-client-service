# Software Bug

## Metadata

```json
{
  "domain": "technical_support",
  "subdomain": "software_bug",
  "type": "subdomain"
}
```

## Description

Issues caused by defects or unexpected behavior in software applications. Includes crashes, incorrect outputs, performance problems, and features not working as intended.

## Common Issues

* Application crashes or freezes
* Features not working as expected
* Incorrect or inconsistent outputs
* Performance degradation or slow response
* UI/UX glitches or display issues
* Errors after updates or deployments
* Compatibility issues across devices or environments

## Root Causes

* Bugs in code logic or implementation
* Unhandled edge cases
* Regression after updates or patches
* Incompatible software versions or dependencies
* Memory leaks or resource mismanagement
* API or integration failures
* Environment-specific issues (OS, hardware, configs)

## Troubleshooting Steps

1. Reproduce the issue consistently
2. Check for known issues or recent updates
3. Collect error messages, logs, and screenshots
4. Verify software version and environment details
5. Restart application or system
6. Clear cache or temporary data
7. Test in a different environment or device
8. Roll back to a previous stable version if applicable
9. Apply patches or updates if available

## When to Escalate

* Issue cannot be reproduced or diagnosed
* Critical functionality is blocked
* Bug affects multiple users or systems
* Security-related vulnerabilities detected
* Requires code changes or developer intervention

## Response Guidelines

* Ask for clear reproduction steps
* Request logs, screenshots, and environment details
* Acknowledge the issue and its impact
* Avoid assigning blame; focus on resolution
* Provide temporary workarounds when possible
* Communicate status updates clearly
